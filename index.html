Accessible Components

Accessible components are those that are designed with the needs of as many users as possible in mind. This includes keyboard navigation, screen reader support, and other accessibility features.

Edited State Diagrams

My revised state diagram for keyboard interaction. I have tweaked the way escape works slightly so that most of the time it will simply close the dropdown outright. This improves learnability, since it aligns more closely with typical escape functionality, but it could also make the interface less efficient for keyboard users who often switch between options in the menu, since it eliminates a method of traversing to the top.

My revised state diagram for mouse interaction makes use of right click for different functionality. While in the original implementation, right and left click both took the user to the page corresponding to the suggestion, right click now removes the suggestion instead. This makes a tradeoff between efficiency and learnability, since it might be confusing for some users who right click only to find their option gone.

My Revised Component

State diagrams represent the flow of inputs and their corresponding output on a component. Here, I'm representing the user interface with one element of the suggestion menu.

My revised component is a dropdown menu that appears when the user types into the search bar. It is designed to be accessible to all users, including those with disabilities. The dropdown menu includes keyboard navigation, screen reader support, and other accessibility features.

In my revision, I have made the following changes:

  • Added keyboard navigation for users who cannot use a mouse
  • Improved screen reader support for visually impaired users
  • Made the dropdown menu more responsive to touch input
